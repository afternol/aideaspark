# BizIdea 事業方針

プロダクトの方向性・ターゲット・差別化・ロードマップを記録する。
機能追加やデザイン判断の根拠として参照すること。

---

## プロダクトビジョン

**起業・新規事業の「何をやるか」の意思決定を科学的にサポートする。**

BizIdea は運営がビジネスアイデアを配信する「メディア型プラットフォーム」。
ユーザー投稿ではなく、AI 分析 + 独自リサーチで構造化されたアイデアを厳選配信する。

## ターゲットユーザー

| セグメント | ニーズ | 利用シーン |
|---|---|---|
| 起業準備中の個人 | 「何をやるか」のアイデア探し | 週末にアイデアを眺めて比較検討 |
| 新規事業担当者 | 市場調査・アイデア候補の情報収集 | 社内提案用の材料探し |
| 副業を始めたい会社員 | 低投資で始められるアイデア発見 | 難易度・投資規模でフィルタリング |
| 投資家・VC | トレンド把握・投資先の着想 | トレンドレーダー・ランキング参照 |

## 差別化ポイント

1. **6軸定量評価**: 新規性・市場規模・収益性・成長性・実現可能性・参入障壁をスコアリング
2. **構造化された情報**: コンセプト・ターゲット・課題・プロダクト・収益モデル・競合・優位性を網羅
3. **トレンド連動**: Google Trends + 業界知見を統合したトレンドスコアリングとアイデアの紐付け
4. **比較・整理機能**: 横並び比較、コレクション管理、メモで意思決定を支援

## サービス名の命名方針

アイデアのサービス名は、事業内容とターゲットに応じてトーンを使い分ける（全部横文字・全部日本語にしない）。

| 事業タイプ | トーン | 例 |
|---|---|---|
| B2B SaaS・テック | 洗練された英語 | MeetScore, LegalEye |
| 消費者向け・生活密着 | 親しみやすい日本語 | みまもりペット, おうちレストラン |
| Z世代・ゲーム感 | キャッチーな造語 | マネクエ, Sleepy |
| 社会課題・福祉 | 信頼感 + カタカナ | ケアFit, リペアマッチ |

**判断基準**: ターゲットが初見で事業内容を想像できるか。

## アイデア配信の方針

- 毎週新規アイデアを追加（目標: 週5〜10件）
- カテゴリは10グループ・80+領域をカバー
- 対象業界は6グループ・35+業界
- 対象顧客は3グループ・25+セグメント
- アイデアには必ず6軸スコア + スコア根拠コメントを付与

## UX 方針

- **一覧 = アクション可能**: カード上でリアクション・コレクション保存ができる。詳細を開かなくても基本操作が完結
- **フィルターは常設**: フィード、ランキング等の一覧系ページには必ず絞り込み手段を用意
- **情報密度のバランス**: カードには「判断に必要な最小限」を表示。飾り情報は削る（例: 難易度バッジ削除）
- **段階的な情報開示**: サマリーカード → 詳細ページ → AI 深掘りの3段階で情報を開示

## マネタイズ戦略

### 収益モデル（検討中）

| プラン | 月額 | AI機能 |
|---|---|---|
| **Free** | 0円 | AI検索月3回、コレクション3フォルダ、コメント閲覧 |
| **Pro** | 1,980円/月 | AI無制限、ビジネスプラン生成、競合分析、キャリアマッチング |
| **Enterprise** | 要相談 | AIメンター、カスタムレポート、API連携、チーム利用 |

### 課金意欲が高い機能（優先度順）
1. **AIビジネスプラン生成**: アイデアを選んでボタン1つで事業計画書の草案が出る。コンサル代の代替
2. **AI競合分析レポート**: SWOT・ポジショニングマップの自動生成
3. **AIキャリアマッチング**: スキル×アイデアの成功確率分析

### AI機能の差別化原則
- 「ChatGPTでもできること」は作らない
- BizIdea独自のデータ（88領域トレンドスコア・6軸評価・構造化DB・ユーザー行動履歴）を必ず活用
- AI出力はプラットフォーム内アクション（比較・保存・プラン生成）に接続する
- 壁打ち単体の価値は薄い。後続アクション（プラン生成→PDF出力→コレクション保存）と統合して初めて価値が生まれる

## 機能の取捨選択方針

### 入れない/削除する判断基準
- 「今のユーザー数・利用頻度で意味があるか」→ Noなら時期尚早
- フォロー/フォロワー: コミュニティ成熟前に入れても使われない → 削除済み
- 「やります」宣言: エンゲージメントとして重すぎ → 軽いリアクション（いいね等）に集約
- 表示項目: 「ユーザーがその場で意思決定に使うか」で判断。飾り情報はノイズ

## PAINTプラットフォーム戦略（2026-04-19 改訂）

### プラットフォーム名
**PAINT = Platform for Augmented Innovation aNd Transformation**
- AideaSpark（旧BizIdea）はPAINTの第1サービスとして位置づけ
- URL: paint-platform.com/aideaspark

### プラットフォーム全体の機能構成（platform-feature-list.xlsx準拠）
```
①発見・機会探索  → /aideaspark（AideaSpark）
②リサーチ・調査  → /research（AIリサーチ、次の実装候補）
③事業設計・計画  → /plan
④バリデーション  → /validate
⑤調達・マッチング → /partner
⑥実行支援       → /aideaspark内に統合
⑦成長・エコシステム → /media + 全サービス横断
🤖 AIエージェント層 → 全サービス横断（最終形）
```

### ドメイン・アーキテクチャ方針
- 単独ドメイン × サブディレクトリ（SEO集約・認証共通化のため）
- Phase1: 単一Next.jsアプリ + basePath
- Phase2: Next.js Multi-zones（サービス増加時）
- DB: Supabase（PostgreSQL）でスキーマ分離
- 将来の分離売却に備えてサービス間境界を明確に設計

### 売却戦略
- プラットフォーム全体売却が最優先（バリュエーション最大化）
- AIエージェント層のデータ統合が競争優位の核心
- 個別売却は交渉長期化時のピボットオプションとして設計上担保
- 目標: ARR3〜5億円 → 50〜100億円バリュエーション
