# learning/

BizIdea プロジェクトの知見・方針・履歴を蓄積するディレクトリ。
Claude Code や開発者が作業前に参照し、同じ失敗の繰り返しや方針のブレを防ぐ。

## ファイル一覧

| ファイル | 内容 | 更新タイミング |
|---|---|---|
| `learnings.md` | 開発で得た再利用可能な学び・教訓 | バグ修正・設計変更・指摘を受けた時 |
| `history.md` | 時系列の開発ログ（何をいつなぜ変えたか） | 機能追加・改修のたびに |
| `development-policy.md` | 技術スタック・設計原則・コーディング規約 | アーキテクチャ変更時 |
| `business-policy.md` | ビジョン・ターゲット・差別化・ロードマップ | 事業方針変更時 |
| `personas.md` | ターゲットペルソナ5名の詳細定義 | ペルソナ追加・見直し時 |

## 使い方

### Claude Code が参照する場合
1. 作業開始前に `development-policy.md` を読み、技術方針に従う
2. 関連する学びが `learnings.md` にないか確認し、同じ問題を繰り返さない
3. 作業完了後、新しい学びや変更を該当ファイルに追記する

### 人間が参照する場合
- プロジェクトの全体像 → `business-policy.md`
- 過去に何をしたか → `history.md`
- 技術的な決定の理由 → `development-policy.md` + `learnings.md`
